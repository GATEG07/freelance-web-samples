# QA Checklist

## Technical checklist

- [ ] Root `index.html` opens correctly.
- [ ] PowerFit overview page opens correctly.
- [ ] Broken version opens correctly.
- [ ] Fixed version opens correctly.
- [ ] Responsive layout checked on mobile.
- [ ] Responsive layout checked on desktop.
- [ ] Browser console has no unexpected errors in fixed version.
- [ ] Navigation links work.
- [ ] Buttons link to the intended pages/sections.
- [ ] Fixed form blocks invalid submissions.
- [ ] Fixed form shows clear validation messages.
- [ ] No external dependencies.
- [ ] No frameworks.
- [ ] No external images.
- [ ] README is clear.
- [ ] GitHub Pages live URL works.

## Commercial checklist

- [ ] Demo is visible from the root page.
- [ ] Client can access broken and fixed versions easily.
- [ ] Before/after value is clear.
- [ ] Screenshots prepared:
  - [ ] Broken mobile navigation
  - [ ] Fixed mobile navigation
  - [ ] Broken form validation
  - [ ] Fixed form validation
  - [ ] Broken services grid
  - [ ] Fixed services grid
- [ ] Explanation is clear for non-technical clients.
- [ ] Scope is closed.
- [ ] Demo is marked as fictional.
- [ ] No fake client claims.
- [ ] No exaggerated promises.
